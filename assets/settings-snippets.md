# settings-snippets.md — ready-to-copy settings.json snippets

This file isn't read by Claude Code — it's reference documentation. Copy the snippets that make sense into the consuming project's `settings.json` (project) or `settings.local.json` (user) and adapt the paths/domains to the real ones.

## 1. Flutter sandbox — `allowWrite` / `allowRead`

A Flutter/Dart workspace needs the command sandbox to allow writes to toolchain caches (outside the repo) and reads of generated artifacts inside the repo — without this, every `flutter pub get`/`build_runner` asks for manual approval.

```json
{
  "sandbox": {
    "enabled": true,
    "filesystem": {
      "allowWrite": [
        "/opt/homebrew/Caskroom/flutter/flutter/bin/cache",
        "~/.dart-tool",
        "~/.pub-cache",
        "~/.flutter"
      ],
      "allowRead": [
        "./**/.dart_tool",
        "./**/.flutter-plugins-dependencies",
        "./**/.flutter-plugins"
      ],
      "denyRead": [
        "~/.ssh",
        "~/.aws",
        "~/.config/gcloud",
        "~/.npmrc",
        "~/.netrc"
      ]
    },
    "excludedCommands": ["gh", "docker"],
    "allowedDomains": [
      "pub.dev",
      "*.pub.dev",
      "storage.googleapis.com",
      "github.com",
      "*.githubusercontent.com"
    ]
  }
}
```

Notes:
- `allowWrite` here is always a path to a **toolchain cache outside the repo** (Homebrew Caskroom, `~/.pub-cache`, `~/.dart-tool`, `~/.flutter`) — never a path inside the project. Adjust the Caskroom path if Flutter isn't installed via Homebrew, or remove the entry if you use `fvm`/another SDK manager.
- `allowedDomains` above is the generic core (pub.dev + GitHub). Add the consuming project's backend/CDN domains — don't inherit domains from another project.
- `denyRead` of credentials (`~/.ssh`, `~/.aws`, etc.) is defense in depth — applies to any project, not just Flutter.

## 2. Codegen deny-list — don't edit generated files

Files generated by `build_runner` (`.g.dart`, `.freezed.dart`, `.config.dart` — json_serializable, freezed, injectable) shouldn't be edited directly: the source of truth is the annotated file that generates them, and a manual edit disappears on the next `build_runner build`. Blocking the *tool* from editing these files keeps the agent from "fixing" an error in the wrong place.

```json
{
  "permissions": {
    "deny": [
      "Edit(./**/*.g.dart)",
      "Edit(./**/*.freezed.dart)",
      "Edit(./**/*.config.dart)",
      "Read(./**/build/**)",
      "Read(./**/.dart_tool/**)",
      "Read(./**/ios/Pods/**)",
      "Read(./**/android/.gradle/**)",
      "Read(./**/coverage/**)",
      "Read(./**/.flutter-plugins)",
      "Read(./**/.flutter-plugins-dependencies)"
    ]
  }
}
```

Notes:
- The `Edit` deny on the three codegen suffixes is the item this snippet exists to document — it removes the biggest source of "fix in the wrong place" in a Dart project with heavy codegen (mobx_codegen, freezed, json_serializable, injectable).
- The `Read(...)` entries for `build/`, `.dart_tool/`, `Pods/`, `.gradle/`, `coverage/` are read noise: large, binary, or generated directories that don't help the agent understand the code and cost context/time if read by mistake.
- `Edit` here blocks the `Edit`/`MultiEdit` tool; if the harness also exposes `Write` for these paths, replicate the entry with `Write(...)`.
