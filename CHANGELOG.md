New `PreToolUse(Edit|Write|MultiEdit)` hook `plugins/core/hooks/model-routing.sh` reads the session model persisted by `session-start.sh` (itself now reading its own stdin for the first time — SessionStart's JSON may carry `model`/`session_id`, written to `${CLAUDE_PLUGIN_DATA}/state/session-model-<session_id>`, mirroring `read-ledger.sh`'s state-dir convention) and, when it matches a synthesis-tier model name fragment (`fable`, kept as a one-line tuple so the tier list grows without touching the logic around it) AND the edited path is a code artifact (`.sh`/`.py`/`.dart`/`.js`/`.ts`/`.json`/`.jsonl`/`.yaml`/`.yml` — never `.md`, never a `/memory/` or `/docs/` path), emits a one-line advisory once per session (marker keyed `session_id`, mirroring `posture-signal`'s marker mechanics). **Advisory, never blocking, by construction**: PreToolUse's `exit 2` blocks the tool call outright — too strong for a strategy preference that isn't a correctness gate — so this hook always `exit 0` and rides `hookSpecificOutput.additionalContext` (`hookEventName: "PreToolUse"`) instead, the same non-blocking channel `posture-signal` uses on `UserPromptSubmit`. Reverted before release: a low-frequency, hand-edited state file is a near-certain staleness risk, and a confidently-wrong hat injected into every session with mechanical authority is worse than no hat at all. The hat concept survives — held and managed operator-side, in memory, not as harness state — and this CHANGELOG keeps the reversal on the record rather than pretending it never happened, in line with the kit's own history of valuing recorded reversals.
